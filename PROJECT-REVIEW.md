# Coach Titan MVP 项目复盘（面试向）

## 1. 项目定位
- 目标用户：健身爱好者（增肌/减脂/维持），不想精确计算但希望有可用指引的人群
- 核心痛点：记录复杂、心理负担重、难以坚持
- 价值主张：用“拳掌法 + 拍照估算”降低记录成本、保持可执行性

## 2. 关键假设（待验证）
- 用户愿意用拍照替代精确称重记录
- 双角度 + 参照物能提高可信度而不会显著降低完成率
- 简短可执行的反馈优于精确数值

## 3. 关键迭代与思路

### 2026-01-27
- 目标：提升拍照识别可信度，降低不确定感
- 动作：补充“光线/距离/倾斜影响识图”的简短提示
- 验证：拍照完成率、识图满意度（1-5 分）
- 影响：待验证

### 2026-01-26
- 目标：降低记录成本，提高完成率
- 动作：双图拍照（俯拍 + 侧面）+ 参照物识别 + Vercel AI Vision 接入
- 验证：完成率、识图准确感知评分、平均记录时长
- 影响：待验证

### 2026-01-20
- 目标：提升首屏可用性与操作清晰度
- 动作：重构底部导航与主页结构（记录/日志/档案）
- 验证：首次记录步骤数、新用户可完成率
- 影响：待验证

### 2026-01-19
- 目标：建立可迭代的 MVP 基础
- 动作：初始化 Web MVP 与部署通路
- 验证：可运行 + 可迭代
- 影响：形成最小闭环

## 4. 当前结论（阶段性）
- 拍照路径可行，但需要更清晰的拍摄指引与反馈提示
- 用户对“可信度”非常敏感，文案与解释会显著影响接受度
- 目前尚缺少用户数据，验证体系需要尽快补齐

## 5. 下一步计划（1-2 周）
- 设计单角度 vs 双角度的完成率对比测试
- 增加拍照后“可信度评分 + 反馈按钮”
- 收集 5-10 位用户的首次体验反馈

## 6. 面试说明话术（可直接使用）
- 我把 MVP 当作验证实验来做：每次更新都会明确目标、动作、验证方法与预期影响。
- 目前聚焦在“降低记录成本”的核心价值，通过拍照估算替代精确计算。
- 下一步重点是完成率与可信度验证，用数据决定是否保留双角度流程。
